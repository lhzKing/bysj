# 方案 C 实施计划：单品码 QR / 标签的双入口访问

> 编写时间：2026-05-23
> 上游需求：本次对话「我之前的产品数据怎么办，我记不清楚批次的，比如说我看见了一个产品，我想要看一下它的二维码」
> 上游已落地能力：本次已完成「生产赋码工作台打印 / 重打 → PrintLabelDialog 弹真实 QR → 手机扫码跳 `/public/traces/<code>`」（commit `1bce61c`）。本计划在此之上扩入口。

---

## 核心目标

让用户从两条路径都能查看 / 打印一个码的 QR，覆盖三种使用场景：

1. 「我记得码不记得批次」——粘个码就能定位
2. 「从追溯列表点进单品详情想看 QR」——单品页一键打开
3. 「正常生产赋码场景」——现有工作台流程保留

**严守安全边界**——公开未登录路径绝不出现任何写链事件入口。

---

## 安全设计（必读，先看再动）

| 风险点 | 缓解 |
|---|---|
| 「查看二维码」按钮出现在公开页 `/public/traces/:code`（未登录可访问） | **绝对禁止**——`TracePublicView.vue` 不加任何按钮 |
| 未登录用户绕过前端直接调 `printTraceCode` API | 后端已有 `@RequirePermission("trace:code:print")` 拦截，前端按钮按权限渲染只是 UX 不是安全 |
| 有 `trace:view` 但无 `trace:code:print` 的角色（如 USER）误触打印 | dialog 按权限切「仅预览」模式，看到的 confirm 按钮直接是「关闭」不调 API |
| 工作台「按追溯码查询」可被用作枚举攻击工具 | 新接口走 `trace:view` 权限 + 后端拦截器；不返回敏感字段（与现有 `/api/trace-codes/{code}` 同口径） |
| QR 内容本身是 URL（指向公开溯源页）泄露什么 | 公开溯源页本就是脱敏视图，QR ≡ URL 已经默认是公开信息，零新增风险 |

**红线**：
- ❌ `TracePublicView.vue` 不加按钮
- ❌ 登录态 `TraceDetail` 详情页上「打印 / 重打」按钮在前端按 `hasPermission('trace:code:print')` 渲染
- ✅ 后端新接口默认走 `trace:view`，无需新权限

---

## 实施清单（按依赖顺序）

### 第一步：后端新增「按追溯码反查」接口（~20 分钟）

**新增**：`GET /api/trace-codes/{traceCode}`

返回单个 `TraceCode` 行的核心字段，让前端工作台直接定位到该码所属批次（若有）。

| 项 | 内容 |
|---|---|
| 权限 | `trace:view` |
| 返回 | `traceCode / batchId / spuId / serialNo / qrPayload / codeStatus / printCount / activatedTime` 等（已有 `TraceAssignBatchCodeResponse` 可复用） |
| 404 | 单品码不存在（注意区分：v11 回填的历史码可能 `batchId` 为 null，前端要兼容） |
| 涉及文件 | `TraceCodeController.java`（如不存在则新建）+ `TraceAssignBatchCodeQueryService` 扩展一个 `findByTraceCode` 方法 |
| 单测 | 新加 1 个 controller 契约测试 + 1 个 service 测试（命中 / 不命中 / 历史无 batchId 三场景） |

### 第二步：前端 `PrintLabelDialog` 加 `view-only` 模式（~15 分钟）

`frontend/src/features/trace/components/PrintLabelDialog.vue` 改动：

- 加 prop `mode: 'print' | 'reprint' | 'view'`（默认 `print`）
- `view` 模式：confirm 按钮文案改「打印（不上链）」或「关闭」（看你设计），点击后**仍可调 `window.print()` 走浏览器原生打印**，但**不 emit confirm 不调 API**
- 标签上「重打」角标只在 `mode === 'reprint'` 时显示

**单测**：smoke 测试加 1 个用例验证 view 模式下 emit('confirm') 不触发

### 第三步：单品详情页加「查看 / 打印标签」按钮（~30 分钟）

`frontend/src/features/trace/views/TraceDetail.vue` 改动：

- 头部 actions 区加按钮「标签 / QR」
- 按钮可见性：永远可见（登录态都可视）；**文案与行为按权限切换**：
  - 有 `trace:code:print` 权限 + 码状态可打印（GENERATED/PRINTED/...）→ 文案「打印标签」，点击弹 dialog `mode=print` 或 `reprint`（按状态切）
  - 无 `trace:code:print` 权限 → 文案「查看二维码」，弹 dialog `mode=view`
  - 终态（VOIDED/SCRAPPED）→ 文案「查看二维码」，强制 `mode=view`，按钮加 tooltip 说明
- 状态判定逻辑复用 `TraceAssignmentWorkbench` 的 `canPrint()/canReprint()` 函数，**抽到 `frontend/src/features/trace/utils/codeActions.js`** 避免重复

**dialog 接收的 code 对象**：从详情页已有的 snapshot 数据组装，缺失 `qrPayload` 时 dialog 内 `qrValueOf()` 已会用 `window.location.origin` 兜底，无需额外接口调用

### 第四步：工作台加「按追溯码查询」输入（~25 分钟）

`frontend/src/features/trace/views/TraceAssignmentWorkbench.vue` 改动：

- 在「按批次 ID 查询」输入旁边加平行的「按追溯码查询」输入框 + 按钮
- 点击：
  - 调新接口 `GET /api/trace-codes/{traceCode}`
  - **有 `batchId`** → 调现有 `loadBatch(batchId)`，加载后高亮 `selectedTraceCode = traceCode`
  - **`batchId` 为 null**（v11 历史回填）→ 直接把这一行塞进 `batchCodes` 单独显示，`batchDetail` 留空，行内操作按钮按码状态启用
- error 处理：404 显示「未找到该追溯码」

### 第五步：路由 & 公开页确认（~5 分钟）

`frontend/src/features/trace/views/TracePublicView.vue` **不动**，复查一遍确认：
- 没有「打印」「查看 QR」「编辑」任何按钮
- 不引入 `PrintLabelDialog`
- 不引入打印相关 API 模块
