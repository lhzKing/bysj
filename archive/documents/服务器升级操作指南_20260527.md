## 0. 上一条命令为什么挂了

你跑的：
```bash
docker compose exec mysql mysql -uroot -p"..." \   --default-character-set=utf8mb4 trace_db -e "source /tmp/m.sql"
                                                ↑ 这里 `\` 后面跟了空格
```

`\` 续行**必须是行末最后一个字符**，后面跟空格的话 shell 会把 `\` 解释为"转义空格"，于是行就在那里断了，`mysql` 只收到 `-uroot -pXXX` 没收到要执行的 SQL，进入交互模式但 stdin 不是 TTY，最后打了一通 help 然后退出。

**解决**：所有命令改成单行，下面给的就是单行版本，可以直接复制粘贴。

## 3. 应用 migrate_v22（给 SUPER_ADMIN 加新权限）

**关键单行命令**（直接复制粘贴，整段一行无换行）：

```bash
cd /opt/1panel/apps/bysj/deploy
docker compose cp ../backend/sql/migrate_v22_demo_seed_master_permission.sql mysql:/tmp/m.sql
ROOT_PWD=$(grep TRACE_DB_ROOT_PASSWORD .env | cut -d= -f2)
docker compose exec mysql mysql -uroot -p"$ROOT_PWD" --default-character-set=utf8mb4 trace_db -e "source /tmp/m.sql"
```

> 把 `ROOT_PWD` 拆成单独一行，避免 `$()` 嵌套 + 长行带来的转义问题。

**期望输出**（最后一行）：
```
result
Migration v22 (demo seed-master-data permission) completed
```

**验证权限确实落库**：

```bash
docker compose exec mysql mysql -uroot -p"$ROOT_PWD" --default-character-set=utf8mb4 trace_db -e "SELECT id, perm_code FROM sys_permission WHERE perm_code LIKE 'trace:data:%';"
```

应该看到 3 行（id 顺序不重要）：
```
trace:data:generate
trace:data:clear
trace:data:seed-master
```

## 4. 重新构建并重启 **后端 + 前端**

> 这次你本地同时改了 backend 和 frontend，两个都要重新 build。它俩无依赖关系，可以并行做。

```bash
cd /opt/1panel/apps/bysj/deploy
docker compose build --no-cache backend frontend
docker compose up -d backend frontend
```

**等服务起来**（backend 约 20-30 秒，frontend 几秒内）：

```bash
docker compose logs -f --tail=20 backend frontend
```

看到 backend 日志里 `Started TraceApplication`、frontend 没报错就 Ctrl+C 退出。

> 关键：权限是 JVM 本地缓存，所以 SQL 落库 + 后端重启**两步缺一不可**。
> 前端改的是 Vue 文件，必须 `--no-cache` 重新打镜像才能让 `npm run build` 拿到新代码。

## 4.5 ⚠ prod profile 必须显式开启 demo 接口

你服务器跑的是 **prod profile**，demo 三件套（seed-master / generate / clear）默认是**禁用**状态。
不开就会返回 `403 当前环境已禁用示例数据管理接口`。

```bash
cd /opt/1panel/apps/bysj/deploy

# 检查是否已有这个变量
grep TRACE_DEMO_DATA_ENABLED .env || echo "TRACE_DEMO_DATA_ENABLED=true" >> .env

# 让 .env 改动生效（必须 up -d，restart 不重新读 .env）
docker compose up -d backend

# 等 backend 重启完
docker compose logs -f --tail=10 backend
# 看到 "Started TraceApplication" Ctrl+C 退出
```

**演示结束后务必改回 false**（见 §11 收尾），避免线上被误调清掉数据。

## 5. 拿 token（在哪儿跑？）

下面 §5–§7 三步是 curl 调 HTTPS 接口，**任何能访问 https://trace.coldhz.codes 的机器都能跑**。
推荐就在刚才那个 SSH 窗口里继续——bash 语法直接能用，token 不用复制粘贴。

```bash
TOKEN=$(curl -s -X POST https://trace.coldhz.codes/api/auth/login -H 'Content-Type: application/json' -d '{"username":"superadmin","password":"superadmin123456"}' | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['token'])")
echo "${TOKEN:0:30}..."
```

应该打印 token 前 30 个字符（`eyJhbGciOiJI...`）。

## 9. 常见问题速查

| 现象 | 原因 | 解决 |
|---|---|---|
| seed-master-data 返回 `403 无权限执行此操作` | v22 migrate 没生效 OR backend 没重启 | 重跑 §3、确认 SELECT 看到 3 行；然后 `docker compose restart backend` |
| seed-master-data 返回 `403 当前环境已禁用示例数据管理接口` | prod profile 默认禁用 demo 接口 | 编辑 `deploy/.env` 加 `TRACE_DEMO_DATA_ENABLED=true`，然后 `docker compose up -d backend` 让环境变量生效；演示完后改回 false |
| generate-sample-data 返回 `400 demo 主数据未就绪` | 没先调 seed-master-data | 先调 §6 |
| generate-sample-data 报 `count 必须在 1 到 500 之间` | count > 500 | .env 加 `TRACE_DEMO_DATA_MAX_GENERATE_COUNT=1000` 后 up -d backend |
| 任意 trace 详情 VerifyBadge 显示红色 | 灌数据时的密钥跟当前 backend 加载的不一致（比如中途换过 keys） | 调 §10 清空 → 重新调 §7 灌 |
| 命令行 `\` 续行不工作 | 复制粘贴时 `\` 后多了空格 / 是 CRLF 换行 | 改成单行，或者用 `ROOT_PWD=...` 这种变量拆分（本指南都是这样写的） |

## 10. 想重新灌一遍数据

如果在 .env 加过 `TRACE_DEMO_DATA_ENABLED=true`，演示完建议改回：
